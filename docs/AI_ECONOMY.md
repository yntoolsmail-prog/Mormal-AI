# Экономика ИИ в CK3 — сводный разбор (как ИИ тратит золото)

Один файл со всеми функциями, значениями и пошаговым примером. Источники —
`vanilla/00_ai.txt`, `vanilla/00_defines.txt`, `vanilla/scripted_values/00_ai_values.txt`,
`.../00_men_at_arms_values.txt`, `.../00_suggestion_values.txt`.

---

## 0. Базовый принцип: кошелёк ОДИН

У ИИ одна казна (одно число золота) и один помесячный баланс. «Военная казна /
развитие / война-сейчас» — это **не сундуки**, а мысленные ярлыки, которыми ИИ
помечает части одной кучи, чтобы решить, что сейчас можно тратить, а что
«отложено». Плюс есть **два типа расходов**:
- **Разовые** (из кучи): найм латников, постройка здания, найм наёмников.
- **Постоянные** (вычитаются из дохода каждый месяц): содержание латников и
  поднятых левий.

## 1. Все значения и функции экономики

### 1.1 Военная казна (резерв на войну) — `00_ai.txt`
| Ключ | Значение | Смысл |
|---|---|---|
| `MIN_WAR_CHEST` | `{25 25 50 100 200 300 400}` | пол казны по ярусам (граф = **50**) |
| `MONTHS_OF_MAINTENANCE_IN_WAR_CHEST` | `18` | цель = 18 мес. содержания армии |
| `PERCENTAGE_INTO_WAR_CHEST` | `0.6` | пока казна не полна — 60% дохода в неё |

**Цель казны = `max(MIN_WAR_CHEST[ярус]; 18 × месячное содержание армии)`.**
У графа без большой армии 18×содержание мало → побеждает пол **50**.

Параллельные скриптовые значения — `00_ai_values.txt`:
| Функция | Значение |
|---|---|
| `cautious_ai_minimum_war_chest_gold` | `50` (+100 герцог / +200 король / +300 имп.) |
| `conqueror_safe_spending_gold` | `2 × максимум казны` (завоеватель копит вдвое) |

### 1.2 Дележ свободного золота — `00_ai.txt`
| Ключ | Значение | Смысл |
|---|---|---|
| `BUDGET_CATEGORY` | Развитие `0.20` / Война-сейчас `0.80` | как делится остаток после казны |
| `BUDGET_CATEGORY_MAX` | `5000` на категорию | потолок; излишек переливается |
| `BUDGET_CATEGORY_SHORT_TERM_MIN` | `{25 25 200 200 400 400 400}` | подушка ликвидности (граф = **200**) |

- **Развитие (LongTerm, 20%)** — постройки, покупка новых латников, проекты.
- **Война-сейчас (ShortTerm, 80%)** — наёмники, срочные военные нужды.
- **Подушка 200** — ИИ держит ~200 свободных «на случай войны прямо сейчас»
  (быстро нанять наёмников); ниже неё неохотно вкладывает в долгое.

### 1.3 Содержание латников как доля ДОХОДА — `00_ai_values.txt`
| Функция | База | Смысл |
|---|---|---|
| `ai_men_at_arms_expense_gold_min` | `0.15` | быстро довести расход на латников до 15% дохода |
| `ai_men_at_arms_expense_gold_ideal` | `0.40` | цель 40% дохода; выше **не покупает** |
| `ai_men_at_arms_expense_gold_max` | `0.60` | выше 60% дохода → **РАСПУСКАЕТ латников** |

Это **только база**. Реальные модификаторы (все из `00_ai_values.txt`, max каждой
функции жёстко ограничен `0.9`):
- **`_min`:** воин +0.1, завоеватель +0.2, **осторожный +0.1**, строитель −0.1,
  директива вассалу «строить латников» +0.5.
- **`_ideal`:** воин +0.1, завоеватель +0.2, строитель −0.2, директива +0.5; и —
  что важно — **бонусы за «значимость» правителя**: король+ в High medieval с
  доходом ≥20 → +0.2; король+ в Late medieval с доходом ≥40 → ещё +0.2 (т.е.
  поздний богатый король целится до `0.4+0.2+0.2 = 0.8`); мелкий правитель
  (< королевства) с доходом ≥30/≥60 → +0.1/+0.1. Кочевые (`herd`) → +0.15.
- **`_max`:** король+ в High/Late medieval → +0.2/+0.2 (потолок роспуска у позднего
  короля доходит до 0.9, а не 0.6); мелкий правитель → +0.05/+0.1; директива +0.1;
  **админ-вассал +0.3**; и спец-правило: для админ/кочевых при `debt_level ≤ 2`
  значение целиком становится `3` (практически не распускают).

👉 Значит «40%/60%» — это про **мелкого графа в раннюю эпоху**. У крупного позднего
короля цель ближе к **80% дохода**, а роспуск — лишь выше **~90%**. Это сильно
меняет картину для больших держав (они держат куда более крупную постоянную армию).

Отдельные параллельные наборы для НЕ-золотых валют (логика та же, числа другие):
`ai_men_at_arms_expense_prestige_*` (племенные, см. §5), `..._treasury_*`
(административное правительство), `..._herd_*` (кочевые), `..._barter_goods_*`.

### 1.4 Сколько латников держать — `00_suggestion_values.txt` + `00_defines.txt`
| Ключ/функция | Значение | Смысл |
|---|---|---|
| `recommended_num_maa_regiments_value` | `ярус − 1` | граф 1, герцог 2, король 3, имп. 4 |
| `REGIMENT_MAA_CAP_BY_TIER` | `{0 1 2 3 4 5 6}` | **мягкий** кап полков по ярусу (граф = **2**) |
| `REGIMENT_MAA_EXTRA_BY_CAP` | `0.2` | +20% содержания на **каждый** полк за **каждый** полк сверх капа |
| `REGIMENT_DEFAULT_STACK_SIZE` | `100` | бойцов в одном «чанке» (stack) |
| `REGIMENT_DEFAULT_MAX_SIZE` | `3` | базово чанков в полку → **300 бойцов** |
| `REGIMENT_MAX_SIZE` | `1000` | абсолютный потолок полка (поздняя игра) |

⚠️ **`REGIMENT_MAA_CAP_BY_TIER` — это НЕ жёсткий лимит числа полков.** Бриф
`REGIMENT_MAA_EXTRA_BY_CAP` прямо говорит: за каждый полк сверх капа содержание
*всех* полков растёт на 20%. То есть граф **может** держать 3+ полка — это просто
становится прогрессивно дороже, и удорожание возвращается в гейты §1.3 (40%/60%
дохода). Прежняя формулировка «граф = 600 бойцов максимум, БОЛЬШЕ НЕЛЬЗЯ» — **ошибка**.

`recommended_num_maa_regiments_value` (= ярус − 1) и `REGIMENT_MAA_CAP_BY_TIER` —
это **разные** вещи: первое — сколько полков ИИ *хочет* (граф 1), второе — после
скольких полков включается **штраф к содержанию** (граф 2).

### 1.5 Цены латников — `00_men_at_arms_values.txt` (найм / содержание мирн.)
| Тип | Найм | Содержание/мес (мирн.) | в войне (×3) |
|---|---|---|---|
| Застрельщики | 45 | 0.15 | 0.45 |
| Лучники | 55 | 0.20 | 0.60 |
| Копейщики | 75 | 0.30 | 0.90 |
| Тяжёлая пехота | 90 | 0.40 | 1.20 |
| Лёгкая конница | 85 | 0.35 | 1.05 |
| Тяжёлая конница | 200 | 0.70 | 2.10 |

`@high_maint_mult = 3` — в поднятом (военном) состоянии содержание ×3.

**Как именно набирается полк (важно, исправлено):** полк состоит из «чанков» по
`REGIMENT_DEFAULT_STACK_SIZE = 100` человек, базово `REGIMENT_DEFAULT_MAX_SIZE = 3`
чанка → **полный полк по дефолту = 300 бойцов** (а не 600 — это была ошибка). Числа
в таблице 1.5 — это базовая стоимость найма/содержания **полка** из
`00_men_at_arms_values.txt`; как именно она масштабируется на число чанков, точно
задаётся в `common/men_at_arms/*` (его у нас нет — это место остаётся
допущением модели, см. §4). Абсолютный потолок размера полка в поздней игре —
`REGIMENT_MAX_SIZE = 1000` (плюс ограничение `REGIMENT_MAX_SIZE_PERCENT = 0.2` от
максимума поднимаемых войск).

### 1.6 Содержание солдат и левий — `00_defines.txt`
| Ключ | Значение | Смысл |
|---|---|---|
| `GOLD_COST_PER_SOLDIER` | `0.003` | золота за **каждого** солдата/мес |
| `RAID_ARMY_COST_MULT` | `0.5` | рейд-армии содержатся вдвое дешевле |

**Про левии (важно, по твоему вопросу):**
- **Поднять левии — бесплатно** (нет цены найма).
- **Но поднятые левии стоят содержания**: `0.003 × число бойцов / мес`.
  - 5000 поднятых левий = `5000 × 0.003 = 15 золота/мес`, пока подняты.
- **Не подняты (сидят дома) — стоят 0.**

То есть мой прежний тезис «левии не стоят денег» верен **только для мира**
(не подняты). В войну поднятые левии — реальный расход, который вместе с
наёмниками и съедает бюджет.

### 1.7 Роспуск устаревших — `00_ai.txt`
| Ключ | Значение | Смысл |
|---|---|---|
| `REGIMENT_OBSOLETION_SCORE_DIFFERENCE` | `20` | распустить полк, если на 20 хуже лучшего И денег нет |

## 2. Куда что идёт (исправлено)

Из **одной** казны ИИ мысленно делает корзины:
- **Войн.казна** — резерв на войну (граф пол 50), не трогать кроме войны.
- **Развитие (20%)** — постройки, проекты.
- **Война-сейчас (80%)** — наёмники И **найм латников** (это военное золото).

Два разных ограничителя на латников:
1. **Золото на найм** берётся из корзины **«Война-сейчас» (80%)**.
2. **Сколько латников держать** упирается в **отсечку по доходу**: содержание
   ≤ `ai_men_at_arms_expense_gold_ideal` (база 40% дохода, у крупных правителей —
   до 80%, см. §1.3). Это **главный и фактически единственный жёсткий стоп** на
   набор.

`REGIMENT_MAA_CAP_BY_TIER` (граф = 2) **не является** жёстким стопом: сверх него
содержание растёт на 20%/полк (`REGIMENT_MAA_EXTRA_BY_CAP`), что просто быстрее
подводит к отсечке по доходу. То есть набор стопает **отсечка по доходу** (40–80%),
а мягкий кап лишь делает «лишние» полки дороже.

И отдельно: если содержание перевалит **`ai_men_at_arms_expense_gold_max`** (база
60% дохода, у крупных — до 90%; упал доход в войну) → **роспуск**.

## 3. Пошаговый пример с корзинами и формулами

> ⚠️ Пример **иллюстративный** и местами опирается на упрощения (поштучный найм
> «по 100», цена здания, разводка корзин — см. §4). Главное: **стоп набора задаёт
> отсечка по доходу** (`_ideal`), а не «жёсткий лимит полков» — в прежней версии
> вывод на МЕС ~67 был ошибочен и ниже исправлен.

Граф: **казна 100, доход +10/мес**, войны нет. Цель — набирать латников до отсечки
по доходу и строить здания **+0.5 / цена 100**. Латники: тяж. пехота, найм **90**,
содержание **0.4/мес** (мирное, на условный чанк 100 — упрощение). Числа цен из
файлов; разводка корзин и сроки — модель (помечено в разделе 4).

Формат строки: **Казна(итого) = [Войн.казна] + [Развитие] + [Война-сейчас]**.

---

**МЕС 0 — Казна 100 = [Войн.казна 50] + [Развитие 10] + [Война-сейчас 40]**
Доход 10. Латников 0. Действие: **ничего**.
```
почему:
• Войн.казна: цель max(50; 18×содержание≈0) = 50 → забирает 50.
• Остаток 50 делится 20/80 → Развитие 10, Война-сейчас 40.
• Найм латника (90) > Война-сейчас (40) → НЕ нанять.
• Здание (100) > Развитие (10) → НЕ строить.
```

**МЕС 7 — Казна 170 = [Войн.казна 50] + [Развитие 24] + [Война-сейчас 96]**
Доход 10 → 9.6. Латников 0 → 100. Действие: **найм 100 чел −90**.
```
почему:
• Война-сейчас 96 ≥ 90 (цена найма) → деньги на найм есть.
• Отсечка 40%: содержание после = 0.4 ≤ 0.40 × доход 10 = 4.0 → можно.
• Лимит полков: 100 ≤ 600 → можно.
• Платёж −90 из «Война-сейчас»: 96 − 90 = 6.
• Содержание +0.4 → доход 10 − 0.4 = 9.6.
```

**МЕС ~18 — Казна ~186 = [50] + [~45] + [~90]**
Доход 9.6 → 9.2. Латников 100 → 200. Действие: **найм 100 чел −90**.
```
почему:
• «Война-сейчас» снова доросла ≥ 90 (по +7.68/мес = 80% от 9.6).
• Отсечка 40%: содержание 0.8 ≤ 0.40 × 9.6 = 3.84 → можно.
• Лимит полков: 200 ≤ 600 → можно.
• −90 из «Война-сейчас»; содержание 0.8 → доход 9.2.
```

**МЕС ~31 / ~43 / ~55 — наймы 3/4/5** (по той же формуле каждые ~12 мес)
Латники 300 → 400 → 500; содержание 1.2 → 1.6 → 2.0; доход 8.8 → 8.4 → 8.0.
```
почему (каждый раз):
• «Война-сейчас» накопила ≥ 90 → платит −90.
• Отсечка 40% всё ещё НЕ достигнута (напр. 2.0 ≤ 0.40×8.0 = 3.2) → набор продолжается.
• Лимит полков ещё не достигнут (≤ 600).
```

**МЕС ~67 — Казна ... [Войн.казна 50] + [Развитие ~125] + [Война-сейчас ~6]**
Доход 7.6. Латников 500 → **600**. Действие: **найм −90**, набор продолжается.
```
почему НЕ останавливается тут (исправлено):
• Отсечка _ideal (40%): содержание 2.4 ≤ 0.40 × 7.6 = 3.04 → по деньгам ещё можно → набирает дальше.
• «Лимит полков» 600 — НЕ жёсткий: сверх капа просто +20%/полк к содержанию
  (REGIMENT_MAA_EXTRA_BY_CAP), что быстрее подводит содержание к отсечке 40%.
• ⇒ набор у графа стопает ИМЕННО отсечка по доходу (когда содержание ≈ 0.40 × доход),
  а удорожание сверх капа лишь приближает этот момент. Грубо: стоп около
  содержания ~3.0/мес (≈0.40 × 7.6), т.е. когда дальнейшие полки перестают влезать в 40%.
  (Будь доход 5: отсечка 40% = 2.0 → стоп раньше; будь это поздний король с _ideal 0.8 → набирал бы вдвое больше.)
```

**МЕС ~99 — Казна ~370 = [50] + [~168] + [~200]**
Доход 7.6 → 8.1. Здания 0 → 1. Действие: **постройка −100**.
```
почему только сейчас:
• Развитие ≥ 100 (цена) → есть на здание. Но Развитие копилось медленно: лишь 20% дохода.
• Подушка: «Война-сейчас» ≥ 200 → ИИ спокоен (войны нет) и разрешает стройку.
• −100 из «Развития»; доход +0.5 → 8.1.
```

**МЕС ~119 — постройка #2** (Развитие снова накопило 100; «Война-сейчас» ≥ 200) →
доход 8.6. Дальше снежок, но медленный (на развитие идёт лишь 20%).

---

### Демонстрация роспуска в войне (отсечка 60%)
Допустим, армия из 600 латников и доход упал в войну 10 → 4, латники подняты
(содержание ×3 = 2.4 × 3 = 7.2/мес):
```
почему роспуск:
• Содержание поднятых 7.2 vs доход 4 → 7.2 / 4 = 180% > 60% (ai_..._gold_max).
• ⇒ ИИ РАСПУСКАЕТ латников, чтобы влезть обратно под лимит.
• Мир → доход 10, содержание мирное 2.4 = 24% < 40% → закупает обратно. Качели.
```

## ИТОГ по двум твоим вопросам
1. **Найм латников оплачивается из корзины «Война-сейчас» (80%-я, военная).**
2. **Набор идёт до отсечки по доходу** (`ai_men_at_arms_expense_gold_ideal`: база
   40%, у крупных поздних правителей — до 80%). Это и есть жёсткий стоп.
   `REGIMENT_MAA_CAP_BY_TIER` — **мягкий** кап: сверх него содержание дорожает на
   20%/полк, что лишь приближает отсечку по доходу, но сам по себе набор не
   останавливает.

## 5. Племенная эпоха: бюджет на ПРЕСТИЖЕ

В племенную эпоху престиж — вторая «валюта»: латники покупаются/содержатся
**престижем**, и **постройки тоже стоят престиж** (по твоим данным ~`70 золота +
200 престижа`). То есть племенник тратит престиж и на армию, и на стройки.

### Сколько престижа КАПАЕТ в месяц (подтверждено — `00_defines.txt`)
`MONTHLY_PRESTIGE_GAIN_PER_TITLE_TIER = { 0 0.125 0.25 0.5 1 2 4 }` (в месяц):
| Ярус | Престиж/мес с титула |
|---|---|
| Барон | 0.125 |
| **Граф** | **0.25** |
| Герцог | 0.5 |
| Король | 1 |
| Император | 2 |
| Гегемон | 4 |

Плюс мелочь: за члена династии `0.02` (кап `2`), за супруга, модификаторы черт/быта.
Итого даже у короля ровный приток престижа ≈ **1–3/мес** — **крошечный**.

### Откуда РЕАЛЬНЫЙ престиж — разовые выплаты (подтверждено)
`CREATE_TITLE_PRESTIGE_GAIN = {0 0 0 300 400 500 600}` (создать герцогство+ титул),
`USURP_TITLE_PRESTIGE_GAIN` — то же; плюс набеги, выигранные войны, заложники,
кровные братья и т.д. Это **сотни престижа разом**.

### Значения престижа (подтверждено файлами)
| Параметр | Значение | Источник |
|---|---|---|
| `ai_men_at_arms_expense_prestige_min` | `0.25` | `00_ai_values.txt` |
| `ai_men_at_arms_expense_prestige_ideal` | `0.40` | — цель: 40% **притока престижа** на содержание |
| `ai_men_at_arms_expense_prestige_max` | `0.60` | — выше → **роспуск** |
| `MAA_PRESTIGE_BUY_COST_MULT` | `2` | найм в престиже ×2 (дороже!) |
| `MAA_PRESTIGE_LOW_COST_MULT` | `0.7` | содержание (мирн.) ×0.7 |
| `MAA_PRESTIGE_HIGH_COST_MULT` | `0.7` | содержание (война) ×0.7 |
| `MAA_STARTING_PRESTIGE_EXPENSE` | `0.4…0.55` | старт: 40–55% престижа на латников |

Итого тяж. пехота в престиже: найм `90×2 = 180` престижа за 100 чел; содержание
`0.4×0.7 = 0.28` престижа/мес за 100.

### Главное отличие — приток лумповый (рывками)

**Механизм тот же, что у золота: проценты считаются от ПРИТОКА (дохода), а не
«весь кошелёк делится заново каждый тик».** Но у престижа приток устроен иначе:
- **Ежемесячный приток престижа мал** (титульный доход престижа невелик).
- **Большая часть престижа — разовые выплаты** (выигранная война, события,
  интеракции, набеги).

Последствие:
- **Содержание латников привязано к 40% от МАЛОГО месячного притока** → постоянная
  армия, которую племенник может *держать*, невелика.
- **Найм — разовая трата из НАКОПЛЕННОГО баланса** престижа. После крупной выплаты
  баланс скачет вверх → племенник нанимает латников **рывком**.
- Но если рывком нанял больше, чем тянет месячный приток (содержание > 60%
  притока) → **роспуск** (та же отсечка `_max = 0.6`).

То есть **армия племенника растёт всплесками после престижных «получек»**, но
потолок «сколько удержать» задаёт скромный месячный приток престижа (правило 40%).
Нет ровного потока — нет стабильной большой армии латников.

### Постройка за `70 золота + 200 престижа` — как считается
Чтобы построить, нужны **обе** валюты одновременно:
- **70 золота** — проходит по золотому бюджету (корзина «Развитие» 20%, порог
  `BUILDING_MIN_SCORE ≥ 0.8`). Это посильно.
- **200 престижа** — а вот это узкое горло. При ровном притоке `0.25/мес` (граф)
  копить 200 — это `200 / 0.25 = 800 месяцев` (нереально). Значит **200 престижа
  берутся только из разовой выплаты** (набег/война/создание титула).

⇒ **Племенные постройки фактически привязаны к престижным «получкам»**: пока не
случилось крупное событие на сотни престижа — здание не построить, даже если
золото (70) есть. Развитие племени идёт **рывками по событиям**, а не ровным
потоком.

### Что я НЕ могу подтвердить по текущим файлам (честно)
Точного правила, **в каком порядке** ИИ тратит накопленный престиж (на латников vs
постройку vs создание титула vs «придержать»), в `00_ai.txt / 00_defines.txt /
00_ai_values.txt` **нет**. Подтверждено только:
- ровный приток престижа (числа выше);
- бюджет содержания латников = % притока престижа (`0.25/0.40/0.60`);
- цена латников в престиже (найм ×2, содержание ×0.7).

Чтобы дать **точный** алгоритм траты престижа на постройки, нужны:
- `common/buildings/*` — точная цена здания (золото+престиж) и его `ai_value`;
- возможный define-резерв престижа (если есть) — пока такого в файлах не нашёл
  (`MIN_RESERVED_*` есть только для золота).

### Чего у престижа точно НЕТ (в отличие от золота)
- **Нет «военной казны» престижа** (`MIN_WAR_CHEST` — только золото).
- **Нет найденного дележа 20/80 для престижа** (`BUDGET_CATEGORY` описан для золота).

## 6. ПРИНЦИП «латники или здание» (главное — найдено в `00_ai_values.txt`)

Вот недостающий **принцип**, как ИИ выбирает между латниками и постройкой. Когда у
ИИ есть деньги на вложение, он делает **взвешенный бросок монетки**: с
соответствующей вероятностью берёт **латников**, иначе — **здание**. Вероятность
зависит от того, **насколько недобран** расход на латников — есть **две разные**
функции (это важно, прежняя версия описывала только вторую):

**Когда расход НИЖЕ минимума (< 15% дохода) → `ai_men_at_arms_chance_expense_below_min`:**
```
  база = 0.40           # по умолчанию 40% латники / 60% здание (агрессивнее добирает)
  + воин +0.40 · завоеватель +0.60 · осторожный +0.20 · строитель −0.30
  + король+ High/Late medieval +0.20 / +0.40
  фокус «строить в столице» → ×0, затем +0.1 (≈0.10)
  целестиальные военные контракты → +1 · директива «строить латников» → +1
  предел: 0 … 0.9
```

**Когда расход МЕЖДУ минимумом и идеалом (15…40% дохода) → `ai_men_at_arms_chance_expense_below_ideal`:**
```
  база = 0.10           # по умолчанию 10% латники / 90% ЗДАНИЕ
  + воин +0.40 · завоеватель +0.60
  + король+ High/Late medieval +0.20 / +0.20 · мелкий правитель +0.05 / +0.10
  × строитель (builder) ×(−0.10)   # строитель почти не берёт латников
  фокус «строить в столице» → ×0, +0.1 (≈0.10, почти всегда здания)
  целестиальные военные контракты → ×0, +0.9 · директива → +1 (всегда латники)
  предел: 0 … 0.9
```

**Что это значит:**
- **Пока расход < 15% (ниже min)** — ИИ относительно охотно добирает латников
  (база 40%). **Между 15% и идеалом 40%** — резко смещается к ЗДАНИЯМ (база 10%
  латники / 90% здания). Латников форсируют воин/завоеватель (+0.4/+0.6).
- Как только латники достигли **идеала** (40%, у крупных — до 80%, §1.3), обе
  функции больше не вызываются — дальше только здания.
- Безземельные (авантюристы) считают по `ai_men_at_arms_chance_landless_ruler`
  (база `0.5`, со своими поправками: лагерь наёмников/бандитов +0.25, учёных −0.25,
  админ −0.3, минимум зданий авантюрера −0.4 и т.д.).
- Само здание-кандидат должно пройти порог `BUILDING_MIN_SCORE_COMPARED_TO_BEST =
  0.8` (`00_ai.txt`) — отбрасываются постройки слабее 80% от лучшей доступной.
