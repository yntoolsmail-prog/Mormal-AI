# CLAUDE.md

Руководство для Claude Code (и для людей), работающих в этом репозитории.

## Что это за проект

**Mormal AI** — мод для **Crusader Kings III** (CK3 — «КС3»/«KC3»).
Цель одна и узкая: **сделать ИИ более конкурентным противником** — чтобы он
лучше вёл войны, точнее рассчитывал размер армий, разумнее выбирал цели и
тратил ресурсы — но не превращался в нечестного чит-бота.

Это мод *на данные/баланс*. Поведение ИИ в CK3 задаётся настраиваемыми
значениями в скриптовых файлах игры (движок Clausewitz, простой текст вида
`ключ = значение`). Здесь нет компиляции и нет кода движка — мы меняем числа и
правила, которые игра уже читает.

Целевая версия игры: **CK3 1.19.x**. Ванильные файлы включают
административное правительство, ярус «Гегемония» и кочевую механику `herd` (стадо).

## Что уже сделано / ключевое знание (держать в голове)

- **Племя платит за латников ПРЕСТИЖЕМ** (`tribal_government`: `regiments_prestige_as_gold = yes`).
  Приток престижа: ярус титула + флэт `monthly_prestige = 0.2` формы + члены династии.
- **РЕАЛИЗОВАНО:** лесенка расхода престижа на латников по запасу престижа →
  `common/script_values/zz_mormal_prestige_values.txt`. Бьёт только племя (только
  оно использует `_prestige_*`). Плейтест: племя держит максимум стеков. ✅
- **Кочевник: вся армия держится на СТАДЕ.** `nomadic_riders` = прямая конверсия
  стада в конницу (костяк); боевые латники орды (`09_mpo_maa_types.txt`) — найм/долив
  подставляются в стадо (`MAA_HERD_*_COST_MULT`), содержание идёт золотом, а золото =
  `GOLD_FROM_HERD_FACTOR (0.0005) × стадо`. Решение «нанять/распустить» считается по
  **золоту** (`_gold_*` со спец-ветками `government_is_nomadic`, max=3 → почти не распускают).
- **Почему степь проседала на «Очень сложно»:** бусты сложности качают всем найм/содержание
  латников, **скорость ополчения** (`levy_reinforcement_rate +3`) и дешёвые доходные
  стройки — но **прирост стада не качают никак**, а у орды нет обычного ополчения (только
  конное от стада). На «Норме» (без ИИ-бонусов) степь доминирует. Лечение — бустить
  **экономику стада** ИИ-кочевника через модификаторы в блоках сложности (см. ниже).
- **Воинственный (`warlike`) ИИ** по дизайну исключён из блока застройки в
  `ai_budget_manipulation_effect` → копит war chest и не строит даже при открытых
  слотах (подтверждённый кейс «король-накопитель»). Отдельная нерешённая тема.

Полный разбор всех вопросов и решений — в **`common/task.md`** (рабочий документ
проекта; блоки `▶ Ответ` сверены по `vanilla/`).

### Правило двух слоёв (важно)

- **`vanilla/`** — слепок оригинальных файлов игры, только для чтения. Нужен,
  чтобы делать diff, смотреть дефолтные значения и ребейзлайнить при обновлении
  CK3. **Никогда не редактируй файлы в `vanilla/`** — это подключённая зависимость.
  (`vanilla/scripted_values/` — локальное имя зеркала; настоящая мод-папка CK3 —
  `common/script_values/`, без «ed».)
- **`common/`** — это сам мод. При загрузке файлы отсюда накладываются поверх
  ванильных. Имена грузятся по алфавиту; поэтому `00_mormal_*` идёт после `00_ai`/
  `00_defines`, а для надёжного «перекрыть всё» используем префикс `zz_`.

Наши переопределения должны быть **тонкими**: только меняемые ключи, не копировать
целый ванильный файл. Исключение — **именованные блоки модификаторов** (см. ниже):
их движок перекрывает целиком, поэтому блок копируется полностью.

### Справочные файлы в `vanilla/` можно дозапрашивать

В `vanilla/` лежит не весь набор игровых файлов. Если для задачи нужен ещё какой-то
ванильный файл — **попроси у владельца репозитория** (ориентируйся по
`vanilla/_GAME_FILE_TREE.txt` — там пути всех файлов игры). Не выдумывай содержимое
ванильных файлов по памяти; механику, зашитую в код движка (а не в текст), честно
помечай как «не выводится из файлов».

## Как CK3 читает эти файлы

- `common/defines/*.txt` — глобальные константы в именованных блоках
  (`NAI = { ... }`, `NArmy = { ... }`, `NDynasty = { ... }`). Позже загруженные файлы
  перекрывают ключи; можно переопределить один ключ внутри блока. **Define нельзя
  условить по уровню сложности** — он один на всю игру.
- `common/script_values/*.txt` — функции-значения (`name = { value = … if = {…} }`).
  Перекрываются по имени (последнее определение побеждает). Идиома точного значения
  по зоне: `multiply = 0  add = X`.
- `common/modifiers/*.txt` — именованные пакеты модификаторов
  (`very_hard_difficulty`, `martial_modifier`, …). Переопределение **заменяет весь
  именованный блок целиком** — сначала скопируй ванильный блок, затем правь.
  Блоки сложности: `<level>_difficulty` применяются к **ИИ** (бонусы), а
  `<level>_difficulty_player` — к **игроку** (штрафы). Бафф «только для ИИ» = в блок
  без суффикса `_player`. Модификатор бьёт лишь тех, у кого есть соответствующий
  ресурс (напр. `herd_gain_mult` действует только на формы со стадом).
- Синтаксис: табы, `#` для комментариев, списки `{ a b c }`, числа с фикс. точкой.
  Над большинством ключей в ванили есть `### Brief:` — читай их.

## Рабочие соглашения

- **Документируй каждое изменение по месту:**
  `КЛЮЧ = <новое>      # ваниль <старое> | чем это помогает ИИ конкурировать`
- **Держи обоснование в синхроне** в `common/task.md` (что изменено, гипотеза,
  результат плейтеста).
- **Меняй по одной теме за раз** (разные популяции — племя/кочевник/феодал — можно
  параллельно, атрибуция не смажется).
- **Никаких скрытых читов.** Лучше заставить ИИ *играть лучше* (умные пороги, полное
  использование реальных ресурсов), чем давать сырые бонусы. Бонус добавляешь —
  огради условием/сложностью и громко задокументируй.
- **Только для ИИ, не для игрока:** баффы кладём в ИИ-блоки (`*_difficulty`, проверки
  `is_ai`), не трогая игрока.
- **Безопасно для мультиплеера:** детерминировано, без `random` в общих файлах.
- **Доверяй плейтесту владельца:** часть механики (валюта латников, herd-стоимости)
  зашита в код и не видна в файлах — его игровые замеры авторитетнее догадок.

## Сборка и релиз

- Локально: `bash tools/build_release.sh` → `dist/Mormal-AI-v<version>.zip`
  (внутри: папка `Mormal-AI/` + `Mormal-AI.mod`; кладётся в `.../CK3/mod/`).
  В архив идёт только то, что грузит игра (без `vanilla/`, `task.md`, README модификаторов).
- Автоматически: пуш тега `vX.Y.Z` → workflow собирает ZIP и прикладывает к GitHub Release.
  Любой прогон (в т.ч. ручной «Run workflow») оставляет ZIP артефактом.
- Версия берётся из `descriptor.mod`. `build/` и `dist/` — в `.gitignore`.

## Ребейзлайн при обновлении CK3

1. Перезапиши изменившиеся ванильные файлы в `vanilla/`.
2. `git diff` по `vanilla/` — увидеть, что поменяла Paradox.
3. Сверь наши переопределения (`common/`), особенно **полные копии блоков
   модификаторов** (`zz_mormal_difficulty_ai.txt`) — они молча откатывают патчевые
   изменения блока.
4. Подними `supported_version` в `descriptor.mod` и `.metadata/metadata.json`.

## Проверка (тестового стенда нет)

1. Загрузи мод с открытым `Documents/Paradox Interactive/Crusader Kings III/logs/error.log`.
2. Чистый лог по нашим файлам/функциям = скрипт парсится.
3. Поведенческая проверка — **наблюдательная**: observer/ускоренные игры, частота
   войн, размеры армий, передел карты против немодного базлайна. Учитывай **уровень
   сложности** (бусты ИИ сильно меняют картину — степь на «Норме» vs «Очень сложно»).

## Быстрая ориентация для новой задачи

- Поведение ИИ через define? → `common/defines/00_mormal_ai.txt` (значение/смысл — грепай `vanilla/00_ai.txt`, `vanilla/00_defines.txt`).
- Лесенки расхода ресурса на латников? → `common/script_values/` (образец: `zz_mormal_prestige_values.txt`); ванильные пороги — `vanilla/scripted_values/00_ai_values.txt`.
- Бафф ИИ, привязанный к сложности? → `common/modifiers/zz_mormal_difficulty_ai.txt` (копия блоков из `vanilla/00_basic_modifiers.txt`).
- Валюта латников по формам / бонусы правительств? → `vanilla/government/00_government_types.txt`.
- Кочевники (стадо, тучность, max_herd, юрты, типы орды)? → `vanilla/scripted_values/09_mpo_values.txt`, `09_mpo_domicile_values.txt`, `vanilla/men_at_arms_types/09_mpo_maa_types.txt`, define-блок herd в `vanilla/00_defines.txt`.
- Как ИИ выбирает здание / двигает бюджет? → `vanilla/buildings/_buildings.info`, `vanilla/scripted_effects/00_ai_budget_effects.txt`.
- Не знаешь, где механика? → `vanilla/_GAME_FILE_TREE.txt` (весь индекс) и `vanilla/scripted_values/_INDEX_all_value_files.txt`.
- Что уже разобрано/решено и план? → `common/task.md`.
