# =============================================================================
#  Mormal AI — буст экономики СТАДА для ИИ-кочевников через уровни сложности
# =============================================================================
#  ПРОБЛЕМА: на "Сложно"/"Очень сложно" ИИ получает бусты, которые косвенно
#  обходят кочевников. В частности `levy_reinforcement_rate` (+1/+3) ускоряет
#  восстановление ОПОЛЧЕНИЯ — но у орды обычного ополчения нет (только конное от
#  стада + ополчение данников), а вся её армия (nomadic_riders + долив латников)
#  держится на СТАДЕ, прирост которого сложность не качает никак. Итог: бустнутые
#  племена/феодалы давят степь, и она не успевает восстанавливать поголовье.
#  На "Нормально" (без ИИ-бонусов) степь, наоборот, доминирует.
#
#  РЕШЕНИЕ: дать ИИ-кочевнику аналог недостающего бонуса — ускорить восстановление
#  ТУЧНОСТИ ПАСТБИЩ (county fertility), которая у кочевника со временем оскудевает
#  (выпас истощает пастбища, а ИИ не умеет это компенсировать). Привязано к сложности.
#
#  ТОЛЬКО ДЛЯ ИИ: блоки `<level>_difficulty` применяются к ИИ (бонусы); игроку идут
#  отдельные `<level>_difficulty_player` — их НЕ трогаем.
#  ТОЧЕЧНО ПО СТЕПИ: `county_fertility_growth_mult` влияет на стадо только у форм со
#  стадом (кочевник/пастух); у феодала плодородие в стадо не идёт — строка-пустышка.
#
#  ВНИМАНИЕ (ребейзлайн): именованный блок модификатора движок перекрывает ЦЕЛИКОМ,
#  поэтому ниже — ПОЛНЫЕ копии ванильных блоков (vanilla/00_basic_modifiers.txt)
#  + одна наша строка. При обновлении CK3 сверять блоки с ванилью, иначе наша
#  копия молча откатит патчевые изменения Paradox в этих блоках.
#
#  Файл "zz_..." грузится ПОСЛЕ "00_basic_modifiers.txt" → перекрывает эти два блока.
# =============================================================================
